# Fixtures

Each fixture is a pair:

```
<name>.pdf | <name>.docx      # the source resume
<name>.expected.json          # the canonical Resume we expect the importer to produce
```

`expected.json` may be a *partial* Resume — the runner merges it over
`createEmptyResume()`, so you only need to declare the fields you want scored.

## Synthetic fixtures

`001-ats-single-column.pdf`, `002-canva-two-column.pdf`, and
`003-word-table-layout.docx` are **real, parseable files** generated by
[`_generate.mjs`](./_generate.mjs) from fictional data (example.com). Their
contents match the sibling `*.expected.json` targets. Regenerate with:

```
node src/importer/__tests__/fixtures/_generate.mjs
```

Each was built to exercise a specific Phase 1/2 hazard:

- **001** — single-column, full-width. Reading order should equal visual order.
- **002** — full-width header + left sidebar / right main column with a real
  gutter. Naive `item.str` concatenation interleaves the columns; layout
  reconstruction must not.
- **003** — Word section headings (`Heading2`) + 2-column dates/content
  **tables** and list bullets (`numPr`). The table pairing must survive into
  `Line.cells`.

Until the Phase 0 stub is replaced by real extraction, scores stay near zero —
that is the expected Phase 0 outcome, not a failure.
